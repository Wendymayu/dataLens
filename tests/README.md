# 测试数据管理

本目录用于管理电商数据库的测试数据生成、导出和导入。

## 目录结构

```
tests/
├── fixtures/          # 固定的测试数据（小规模，用于单元测试）
├── data/             # 生成的大规模测试数据记录
│   ├── users.json
│   ├── products.json
│   ├── orders.json
│   └── summary.json
└── README.md

scripts/
├── generate_test_data.py    # 测试数据生成脚本
├── export_data.py           # 数据导出脚本
└── import_data.py           # 数据导入脚本
```

## 使用方法

### 1. 生成测试数据

生成大量测试数据并插入数据库：

```bash
# 默认配置：500用户 + 500商品 + 10000订单
python scripts/generate_test_data.py

# 自定义数量
python scripts/generate_test_data.py --users 1000 --products 800 --orders 20000
```

**参数说明**：
- `--users`: 用户数量（默认500）
- `--products`: 商品数量（默认500）
- `--orders`: 订单数量（默认10000）
- `--config`: 配置文件路径（默认config.json）

**生成的数据**：
- 用户数据保存到 `tests/data/users.json`
- 商品数据保存到 `tests/data/products.json`
- 订单数据保存到 `tests/data/orders.json`
- 统计信息保存到 `tests/data/summary.json`

### 2. 数据导出

导出当前数据库数据到JSON文件：

```bash
python scripts/export_data.py
```

### 3. 数据导入

从JSON文件导入数据到数据库：

```bash
python scripts/import_data.py
```

## 测试数据特点

### 用户数据
- 真实的中文姓名、邮箱、手机号
- 会员等级分布：普通60%、VIP25%、黄金10%、钻石5%
- 注册时间：过去2年内随机分布
- 账号状态：95%正常、5%冻结

### 商品数据
- 7个分类：手机通讯、电脑办公、数码配件、男装、女装、休闲食品、酒水饮料
- 真实的品牌和商品名称
- 价格区间符合实际（手机1000-8000元、食品20-200元等）
- 库存和销量随机分布
- 商品状态：90%上架、10%下架或缺货

### 订单数据
- 订单时间：过去1年内随机分布
- 订单状态分布：
  - 待支付：10%
  - 已支付：15%
  - 已发货：20%
  - 已完成：50%
  - 已取消：5%
- 每个订单1-5个商品
- 订单金额：商品单价 × 数量的总和

## 适合测试的NL2SQL查询

生成数据后，可以测试以下自然语言查询：

### 基础查询
- 有多少个用户？
- VIP会员有多少人？
- 商品总数是多少？
- 哪些商品缺货了？

### 统计分析
- 本月订单总额是多少？
- 销量最高的10个商品是什么？
- 哪个用户消费最多？
- 每个分类下有多少商品？

### 复杂查询
- 4月份已完成的订单平均金额是多少？
- 哪些用户在最近30天内没有下单？
- 电子产品分类下价格超过5000元的商品有哪些？
- 订单金额超过1000元的VIP会员有多少人？

### 时间范围查询
- 2024年第一季度的订单总数？
- 最近7天的新增用户数？
- 3月份销售额最高的商品？

### 关联查询
- 购买过iPhone的用户还买了什么？
- 订单数量最多的前10个用户？
- 哪些商品从未被购买过？

## 注意事项

1. **数据量控制**：生成大量数据需要时间，建议从小规模开始测试
2. **数据库性能**：10000+订单会产生25000+订单明细，注意数据库性能
3. **数据一致性**：脚本会自动处理外键关系，确保数据一致性
4. **重复运行**：重复运行会追加数据，如需重置请先清空表
5. **备份数据**：生成的JSON文件可用于数据恢复和迁移

## 清空数据

如需重新生成数据，先清空表：

```sql
USE ecommerce;
SET FOREIGN_KEY_CHECKS = 0;
TRUNCATE TABLE order_items;
TRUNCATE TABLE orders;
TRUNCATE TABLE products;
TRUNCATE TABLE users;
SET FOREIGN_KEY_CHECKS = 1;
```

或使用脚本：

```bash
mysql -h 127.0.0.1 -u root -p52wendyma ecommerce < scripts/clear_data.sql
```
